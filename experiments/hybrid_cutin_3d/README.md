# 하이브리드 교전 컷인 A/B 스파이크

## 목적

지휘관 1명+병사 8명씩을 2D로 유지하면서, 아주 작은 교전 배경만 3D로 렌더링했을 때 공간감과 가독성이 개선되는지 전체 2D 폴백과 비교한다. 이 경로는 승인된 실험이며 정식 전투 시스템에 자동 채택되지 않는다.

## 계약

- 실행 씬: `res://experiments/hybrid_cutin_3d/hybrid_cutin_ab_spike.tscn`
- 입력: `Tab`으로 A/B 배경 전환, `Enter` 또는 `Esc`로 건너뛰기
- API: `play_clash(payload)`, `skip()`, `presentation_finished(payload)`
- 내부 3D 렌더: 640×360 `SubViewport`, 고정 직교 `Camera3D`, nearest 확대
- 캐릭터·접지 그림자·타격 VFX·UI: 바깥 `Control`의 2D draw layer
- 폴백: 3D 비활성·실패 시 동일 payload를 전체 2D 무대로 표시

## 격리 제한

- 3D 배경은 전투 계산, 충돌, 물리, 내비게이션을 소유하지 않는다.
- `Sprite3D`나 3D 캐릭터는 사용하지 않는다.
- 동적 그림자, 후처리, PBR 강조와 외부 애셋을 사용하지 않는다.
- 정식 통합 전 1280×720·1280×800 화면, 반복 재생 성능과 Steam Deck 성능을 별도 검증한다.

## A/B 평가 항목

1. 2D SD 캐릭터 실루엣과 병사 수를 즉시 읽을 수 있는가.
2. 3D 지면의 원근과 프롭이 캐릭터를 가리지 않는가.
3. 2D 폴백과 동일 payload·완료 신호·건너뛰기 결과를 유지하는가.
4. 첫 실행과 반복 실행에서 눈에 띄는 셰이더 컴파일 끊김이 없는가.
5. 3D 미니 필드가 추가 제작·검증 비용을 상쇄할 만큼 화면 품질을 높이는가.

## 현재 한계

- 기본 도형과 코드 드로잉 플레이스홀더라 최종 도트 아트 품질을 대표하지 않는다.
- 헤드리스 테스트는 렌더 구성과 계약을 검사하지만 체감 프레임·화면 조화는 판정하지 못한다.
- 채택 여부는 GUI 캡처와 대상 기기 성능 비교 뒤 사용자 결정 게이트로 올린다.
