# 주몽 UI 시스템

## 소유 영역

- `scenes/ui/`
- `scripts/ui/`
- `docs/ui/`
- UI 전용 그래픽 요구사항

## 구현 원칙

- `Control`과 `CanvasLayer` 기반의 전체 2D UI를 사용한다.
- UI는 데이터를 표시하고 명령을 전달하며 전투·서사 규칙을 직접 소유하지 않는다.
- 직접 결합보다 타입이 명확한 시그널과 이벤트를 사용한다.
- 키보드·게임패드 포커스, 기준 해상도, 스케일링과 가독성을 함께 검토한다.

## 작업 절차

1. 화면 진입·이탈과 사용자 흐름을 먼저 정의한다.
2. 데이터 계약과 신호 소유자를 확인한다.
3. 레이아웃, 상태, 오류·빈 데이터 처리를 구현한다.
4. 화면 캡처가 가능한 승인 기준을 작성한다.
5. GUI 스모크 테스트와 `$jumong-verify` 검증을 요청한다.

## 협업 경계

- 공통 저장·설정 서비스는 `$jumong-core-systems`가 소유한다.
- 전투 상태와 계산은 `$jumong-combat`가 소유한다.
- 원본 UI 이미지는 `$jumong-2d-art`가 소유한다.
