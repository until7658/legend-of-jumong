# 주몽 전투

## 소유 영역

- `docs/combat/`
- `data/combat/`
- 향후 `scenes/combat/`
- 향후 `scripts/combat/`

## 구현 원칙

- 전투 흐름을 명시적 상태머신으로 구성한다.
- 스킬·상태이상·수치는 데이터와 로직으로 분리한다.
- 정적 타입과 시그널을 사용하고 UI 노드를 직접 조작하지 않는다.
- 데미지 공식, 난수 범위, 턴 규칙을 구현 전에 문서화한다.
- 전투 무대와 연출은 2D 노드와 2D VFX만 사용한다.

## 작업 절차

1. 상태 다이어그램과 입출력 계약을 제시한다.
2. 데이터 스키마와 기본 공식을 확정한다.
3. 최소 전투 루프부터 실행 가능한 단위로 구현한다.
4. 구현 담당이 자기 영역의 자동·수동 테스트를 작성한다.
5. 턴 로그와 화면 증거를 `$jumong-verify`에 전달한다.

## 협업 경계

- HUD와 명령 화면은 `$jumong-ui-systems`가 소유한다.
- 캐릭터 액터·애니메이션은 `$jumong-character`가 소유한다.
- 전투용 이미지·VFX는 `$jumong-2d-art`가 소유한다.
