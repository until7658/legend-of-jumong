# 주몽 코어 시스템

## 소유 영역

- `project.godot`
- `scenes/bootstrap/`
- `scripts/bootstrap/`
- `scripts/systems/`
- 공통 설정·저장 데이터 구조

## 작업 절차

1. 호출자와 소비자를 확인하고 최소 공용 인터페이스를 설계한다.
2. 도메인 데이터와 전역 서비스의 책임을 분리한다.
3. 정적 타입, 명시적 시그널, 버전 가능한 저장 형식을 사용한다.
4. 기존 실행 흐름을 유지하며 작은 단위로 통합한다.
5. 메인 씬과 관련 기능의 회귀 검증을 요청한다.

## 2D 게임플레이 및 하이브리드 연출 기준

- 메인 게임 흐름은 2D 씬을 대상으로 구성한다.
- 전투 계산·충돌·내비게이션에 3D 의존성을 추가하지 않는다.
- 승인된 교전 컷인 배경은 전용 SubViewport에 격리하고 2D 폴백·동일 payload·성능 검증을 유지한다.

## 협업 경계

- 개별 UI 화면은 `$jumong-ui-systems`가 소유한다.
- 맵 내부 동작은 `$jumong-level-2d`가 소유한다.
- 서사·전투 규칙을 전역 서비스에 직접 삽입하지 않는다.
