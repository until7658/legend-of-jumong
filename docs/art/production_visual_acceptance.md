# 오프닝·제1장 프로덕션 비주얼 승인 기준 v1

## 목적과 정본

이 문서는 기획 의도와 실제 제작물 사이의 승인 계약이다. 캐릭터·맵·UI·컷신 담당은 아래 정본을 공통 입력으로 사용하며, PM 승인 전 `gold`, `final`, `production_ready`라는 표현을 사용하지 않는다.

- 서사 정본: `docs/narrative/prologue_production_script.md`, `docs/narrative/chapter_01_production_script.md`
- 스타일 정본: `docs/art/unified_visual_style_bible.md`
- 타이틀 정본: `카툰형 02`, `assets/ui/title/keyvisual_dawn_river_stylized_02.png`
- 인물 기준: **17세 주몽**, 젊고 마른 궁수, 적갈색 상의·청흑색 하의, 높은 검은 묶음머리
- 핵심 문장: 거친 먹선과 2.5단 셀 채색으로 그린 고대 북방 판타지 카툰
- 서사 원칙: 인간의 선택과 정치적 불안을 우선하고 초자연 현상을 사실로 확정하지 않는다.

## 공통 하드 게이트

다음 중 하나라도 실패하면 분야별 장점과 관계없이 `거절`한다.

1. 특정 작품·배우·작가의 얼굴, 복식, 로고, 구도 또는 문양을 모사하지 않는다.
2. 조선 관모·단청·흉배, 중국 황제식 면류관, 유럽 판금갑옷, 의미 없는 룬·유사 한자를 쓰지 않는다.
3. 캐릭터와 소품의 생성형 오류(손, 활, 시위, 화살, 여밈, 접지, 알파 프린지)가 최종 화면에 남지 않는다.
4. 순백·순흑 압착, 사진 질감, 균일한 에어브러시 그라데이션, 과도한 광택을 사용하지 않는다.
5. 화면 주역은 배경과 최소 20%의 국소 명도 차를 확보하고, UI 본문은 4.5:1, 큰 제목은 3:1 이상의 대비를 확보한다.
6. 실제 Godot 직교 카메라의 1280×720 무보정 캡처와 100%·75%·50% 축소본으로 판정한다.
7. AI 생성 애셋은 도구·날짜·목적·수정 여부·사용 위치·권리 검토를 `docs/assets/ai_asset_log.md`에 남긴다.

## 분야별 골든 승인 기준

### 캐릭터 원화와 초상화

- 주몽은 17세로 읽혀야 한다. 성숙한 20대 장군 체격, 수염, 과도한 왕권 장식은 거절한다.
- 원화는 7.5~8등신, 필드 변환본은 머리 8~12%, 손·무기 10~15%를 확대한다.
- 얼굴·손·복식 여밈·활과 화살통 결합을 사람이 교정한 정면·측면·후면 턴어라운드가 있어야 한다.
- 기본색, 큰 그림자, 접촉 암부의 2.5단 채색을 유지하고 질감은 표면의 10~20%에 제한한다.
- 주몽의 적갈색은 영웅의 신성 표식이 아니라 플레이 가독성과 인물 식별을 위한 색으로 사용한다.

### 필드·전투 스프라이트

- 프레임은 384×384 투명 PNG, 피벗 `(192, 340)`을 기준으로 한다.
- 일반 탐색 유효 신장은 110~155px, 전투는 145~210px이며 머리 폭 22px, 활 두께 2px 이상을 확보한다.
- 모든 프레임의 발 접점 ±2px, 머리 높이 ±3px, 활 길이·팁 형태 ±1%를 지킨다.
- 활·시위·화살·화살통은 몸체와 분리한 고정 레이어를 사용한다. 좌우 방향은 단순 반전하지 않는다.
- 대기는 호흡이 읽혀야 하고, 걷기는 접촉·하강·통과·상승, 사격은 준비·노킹·만작·릴리스가 같은 골반과 접지에서 이어져야 한다.
- 밝은색·어두운색·녹색 배경에서 알파 가장자리와 녹색 프린지가 없어야 한다.

### 맵·Terrain·오브젝트

- 직교 카메라 수직각 45~50도를 기준으로 캐릭터 상체와 얼굴 방향이 읽혀야 한다.
- 256×256 셀에는 2~4개의 큰 형태군만 두며 사진식 풀·자갈 노이즈는 쓰지 않는다.
- 풀·흙·얕은 물·물가 전환은 Terrain peering 규칙과 4×4 반복 미리보기를 제공하고 경계가 보여서는 안 된다.
- 나무·바위는 투명 배경, 하단 중앙 피벗, 캐릭터보다 가늘고 저채도인 유색 외곽선을 사용한다.
- 플레이 경로와 캐릭터 주변 24~40px는 잡음을 낮춘다. 접지 그림자는 20~38% 불투명도로 하반신을 가리지 않는다.
- 프롤로그 강가는 청회색 새벽, 제1장 훈련장은 공개적이고 건조한 흙 공터, 사냥터는 북쪽 능선의 압박감을 보여주되 마법광을 쓰지 않는다.

### UI·타이틀·컷신

- 타이틀의 기준 이미지는 `keyvisual_dawn_river_stylized_02.png`이며 사실주의 러프는 구도 참고 외 사용하지 않는다.
- 제목과 메뉴 안전 영역은 좌우 7.5%, 상하 6%를 확보한다. 메뉴 뒤에는 인물·배·강한 물결·고대비 실루엣을 두지 않는다.
- 먹청·회청·회갈을 기본으로 호박·적동 강조 면적은 화면의 8% 이하로 제한한다.
- 빛은 자연광이어야 하며 인물이나 배를 선택·추적하는 것처럼 연출하지 않는다.
- 모션 감소 설정을 위한 정지 대체본을 제공하고, 움직이는 안개·반사가 텍스트 대비를 떨어뜨리지 않는다.

## 제출·승인 절차

1. 담당자는 정본 문서와 장면 체크포인트를 작업 브리프 첫 줄에 기록한다.
2. 캐릭터 1포즈, Terrain 4×4, 나무·바위 각 1종, UI 한 화면만 골든 후보로 제출한다.
3. 밝은색·어두운색·녹색 배경, 회색조, 1280×720 통합 캡처를 함께 제출한다.
4. PM은 `승인`, `조건부 승인`, `재작업`, `거절` 중 하나와 실패 항목을 기록한다.
5. `승인` 전에는 동일 계열 양산을 시작하지 않는다. 기능 구현에는 명시적으로 표시된 플레이스홀더만 사용한다.
6. Godot 임포트·씬 연결 뒤 `jumong-verify`가 화면·로그·종료 코드로 별도 판정한다.

## 주요 위험과 의존성

- 캐릭터 골든과 Terrain 골든이 아직 실패 상태라 실제 제1장 통합 화면은 플레이스홀더다.
- 생성형 단일 시트를 프레임·오브젝트로 자동 분리하면 프린지와 구조 불일치가 재발한다.
- 내러티브 이벤트 ID와 맵 ID가 바뀌면 오디오·컷신 트리거도 함께 버전 관리해야 한다.
- 상업 배포 전 AI 애셋·음원·폰트 권리와 Steam 최신 AI 콘텐츠 공개 정책을 재검토한다.
