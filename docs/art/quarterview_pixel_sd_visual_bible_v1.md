# 쿼터뷰 SD 도트 비주얼 바이블 v1

## 정본 선언

- 게임 전체는 2D로 제작한다.
- 필드와 전술 맵은 지면을 45~50도 내려다보는 쿼터뷰다.
- 인게임 캐릭터는 2~4등신 범위, 기본 약 3등신의 도트 캐릭터다.
- 고해상도 대화 초상화·키 비주얼과 인게임 도트 스프라이트는 별도 제작물이다.
- 이전 6등신 비픽셀 필드 캐릭터 방향은 폐기하며 신규 제작 기준으로 사용하지 않는다.

## 픽셀 제작 기준

- 목표 인게임 셀: 기본 64×96px 안에서 약 48~80px 높이. 최종 수치는 4방향 테스트 뒤 고정한다.
- 기본 방향: 남·북·동·서 4방향. 대각선은 M4 이후 비용과 가독성을 비교한다.
- 기본 프레임: 대기 2~4, 걷기 6, 기본 공격 6~10, 피격 3, 제압 4 이상.
- 제한 팔레트와 큰 명암 덩어리를 우선하며 자동 안티앨리어싱과 서브픽셀 이동을 피한다.
- Godot는 Nearest 필터, 픽셀 스냅, viewport 정수 배율을 사용한다.
- 정렬 피벗은 발 중앙이며 무기·화살통·깃발의 방향별 고정점을 매니페스트에 기록한다.

## 쿼터뷰 판정

- 머리 윗면, 어깨선, 발 앞 지면이 동시에 보여야 한다.
- 정면 초상처럼 수평으로 보이거나 완전 탑다운으로 머리만 크게 보이면 실패다.
- 타일의 논리 사각 격자는 화면에서 다이아몬드로 투영하고, 캐릭터 발은 셀 중심보다 약간 아래에 접지한다.
- 겹침 순서는 Y-sort와 발 피벗으로 결정한다.

## 전투의 세 연출 레이어

### 전술 맵

- 분대는 지휘관 1명과 병사 6~12명으로 구성하며 M4는 병사 8명을 표시한다.
- 한 셀의 선택 단위로 움직이되 지휘관은 선두, 병사는 2열 축약 대형으로 읽히게 한다.
- 선택·이동·공격 범위가 병사 실루엣보다 우선해 읽혀야 한다.

### 교전 컷인

- 공격 확정 뒤 별도 짧은 2D 무대에서 양쪽 지휘관과 다수 병사가 대열로 충돌한다.
- 병종, 병사 수, 지휘관 기술, 사기 변화를 1~3초 안에 보여 준다.
- 참고 게임의 좌우 배치, 초상 프레임, HUD 장식, 타격 구성은 복제하지 않는다. 우리 프로젝트는 북방 흙길·초원·강변 무대, 군기, 북소리, 활의 일제사격으로 차별화한다.

### 개인·서사 컷신

- 궁술대회, 개인 결투, 대화, 구조, 즉위 같은 장면은 분대전 연출과 분리한다.
- 궁술대회는 주몽 개인의 호흡, 손가락, 활의 휨, 화살 궤적, 과녁과 관중 반응을 중심으로 한다.
- 필요하면 도트 캐릭터와 고해상도 컷인을 혼합할 수 있지만 동일 인물의 복식·색·무기는 일치시킨다.

## M4 후보 판정

`assets/characters/jumong/pixel_golden_v1/jumong_pixel_idle_south_v1.png`는 SD 비례와 픽셀 덩어리 확인용 승인 후보다. 네이티브 셀 크기, 4방향, 장비 연속성, 애니메이션이 없어 최종 애셋은 아니다.
