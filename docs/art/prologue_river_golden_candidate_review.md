# 오프닝 강변 P0 골든 후보 검토 v1

## 대상과 생성 경로

- Terrain 소스: `assets/tiles/golden_candidates/prologue_river_terrain_candidate_source_v1.png`
- Terrain 정규화 후보: `assets/tiles/golden_candidates/prologue_river_terrain_candidate_v1.png`
- 소품 크로마키 소스: `assets/objects/golden_candidates/prologue_river_props_candidate_source_v1.png`
- 소품 RGBA 후보: `assets/objects/golden_candidates/prologue_river_props_candidate_v1.png`
- 입력 참고: `assets/maps/golden_candidates/prologue_river_asset_sheet_candidate_v1.png`, 카툰형 02 타이틀, `data/maps/map_asset_manifest.json`
- 생성 방식: OpenAI built-in image generation, 2026-07-17. 기존 파일은 덮어쓰지 않았다.

Terrain은 모델 원본 1254×1254를 1024×1024로 bicubic 정규화했다. 소품은 모델 원본 1254×1254의 단색 `#ff00ff` 배경을 로컬 flood-fill chroma-key 방식으로 제거했다. 설치된 `remove_chroma_key.py`는 이 실행 환경에서 Python 런타임을 찾지 못해 실행할 수 없었으며, 같은 목적의 로컬 픽셀 처리와 엄격한 잔류 마젠타 제거를 적용했다. 이 차이는 승인 이력에 남긴다.

## Terrain 검사

### 충족

- 최종 크기 1024×1024, 정확한 4×4 분할과 256×256 셀 위치를 확보했다.
- 흙·풀, 얕은 물, 수평·수직 물가, 안·바깥 모서리와 여울 의도가 육안으로 구분된다.
- 카툰형 02의 회청색 물·회갈색 흙·저채도 녹색과 제한된 호박색을 따른다.
- 물 기본 셀의 반대편 가장자리 평균 RGB 차이는 6.9~8.4로 비교적 안정적이다.

### 미충족

- 생성 이미지가 셀 사이에 어두운 구획선을 그려 넣었다. 셀을 잘라 반복하면 이 선이 타일 격자로 보이므로 무봉합 하드게이트를 통과하지 못한다.
- 흙 기본 셀의 좌우 가장자리 평균 RGB 차이는 11.6~13.2지만 상하 차이는 16.1~16.2다. 검수 임계 15를 초과한다.
- 각 전환 셀의 W/N/E/S peering 의도는 읽히지만, 모든 조합을 완결한 Terrain bitmask는 아니다.
- 미세 돌·풀 빈도가 목표인 셀당 큰 형태군 2~4개보다 많아 캐릭터 주변 시각 휴지 기준을 보장하지 못한다.

### 판정

`재작업 후보 / 게임 연결 금지`. 팔레트와 물가 전환 구도는 후속 수작업 Terrain의 참고로 보존한다. 셀 구획선 제거, 상하 edge paint-over, 디테일 40% 이상 감축, WNES peering 표 작성과 3×3 반복 캡처가 필요하다.

## 소품 알파·분리 검사

### 기계 검사

- 크기: 1254×1254 RGBA
- 투명 픽셀: 1,244,560
- 불투명 픽셀: 327,956
- 네 모서리 alpha: 모두 0
- 엄격 기준 잔류 마젠타: 최초 342px 검출 후 추가 제거, 재검 기준 0px 목표
- quadrant bounding boxes:
  - 나무: `(129,40)–(626,609)`
  - 바위: `(627,207)–(1147,595)`
  - 갈대: `(139,642)–(626,1178)`
  - 작은 배: `(627,820)–(1194,1164)`

네 오브젝트는 서로 다른 quadrant에 있고 bounding box가 겹치지 않아 개별 분리가 가능하다. 나무·바위·갈대는 접지부 중앙을 기준으로 피벗을 잡을 수 있다.

### 미충족

- 로컬 제거 결과가 사실상 binary alpha라 가는 잎·가지의 가장자리를 밝고 어두운 배경에서 다시 확인해야 한다.
- 나무 아래 작은 돌과 바위 주변 파편이 접지 최저점을 넓혀 자동 bottom-center 피벗과 충돌 실루엣을 흐린다.
- 배의 노가 선체 아래로 돌출되어 이미지의 최저점과 선체 접지 기준이 다르다. 배와 노를 분리하지 않으면 일관된 피벗을 만들 수 없다.
- 소품 시트 자체에는 개별 PNG, pivot 좌표, 충돌 폴리곤이 없다.

### 권장 분리 피벗

| 오브젝트 | 분리 후 피벗 원칙 | 충돌 원칙 |
| --- | --- | --- |
| 나무 | 줄기 밑동 중심. 주변 작은 돌 제외 | 밑동 타원만 |
| 바위 | 주 바위군 하단 중심. 파편 제외 | 주 덩어리의 하부 70% |
| 갈대 | 줄기군이 모이는 하단 중심 | 없음 또는 작은 통과 가능 영역 |
| 작은 배 | 선체 용골의 하단 중심 | 선체 외곽. 노는 별도 오브젝트 |

### 판정

`분리 가능한 재작업 후보 / 게임 연결 금지`. 네 종류의 형태·스타일 방향은 P0 후보로 유효하지만 수동 개별 분리, 노 분리, 파편 정리, soft alpha edge, 밝은색·어두운색·녹색 배경 합성 후에만 승격할 수 있다.

## 종합 판정과 다음 승인 게이트
