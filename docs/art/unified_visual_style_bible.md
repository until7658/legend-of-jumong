# 주몽신화전기 통합 비주얼 스타일 바이블 v1

## 1. 목적과 최종 방향

이 문서는 캐릭터 원화, 2D 캐릭터, HD-2D 배경, 타이틀과 UI가 한 작품으로 보이게 만드는 공통 제작 기준이다. 현재 프로젝트의 목표 화풍은 **사실적 역사화가 아니라, 거친 선화와 절제된 셀 채색을 가진 북방 고대 판타지 카툰**이다.

핵심 문장:

> 먹선으로 기록한 인간의 역사 위에 새벽빛과 바람이 신화의 여운을 더한다.

- 기준 인물 언어는 `assets/concept/cartoon_v1/jumong.png`의 날카로운 실루엣, 거친 선, 제한된 셀 명암과 적동 림라이트다.
- 얼굴과 의상은 읽기 쉬운 카툰 형태로 단순화하되 재료의 사용감과 북방 생활성은 유지한다.
- 배경과 타이틀도 사진적 사실주의가 아니라 같은 붓과 안료로 그린 듯한 형태 단순화와 선 선택을 적용한다.
- 특정 게임, 애니메이션, 작가, 드라마의 고유 화풍·인물·의상·구도·문양을 모사하지 않는다.
- 원사료에서 확인되는 인명·지명과 생활 요소를 우선한다. 확인이 어려운 복식·문양·건축은 게임 각색임을 기록하며 중국 왕조식 장포, 조선 관모, 유럽 판금갑옷을 혼합하지 않는다.

## 2. 공통 렌더링 언어

### 2.1 선

- 최종 화면 1080p 기준 인물 외곽선 체감 굵기는 1.5~3 px, 내부선은 외곽선의 45~65%로 한다.
- 선은 완전한 벡터 균일선이 아니라 시작과 끝이 가늘어지는 붓펜 압력을 사용한다.
- 얼굴·손·무기 구조에는 날카롭고 정돈된 선을, 천 끝·머리카락·풀·안개에는 끊기고 마른 붓선을 쓴다.
- 배경의 모든 사물에 검은 외곽선을 두르지 않는다. 플레이 경계, 큰 바위 전면, 나무줄기와 주요 건축 윤곽에만 저채도 유색선을 선택한다.
- 작은 스프라이트에서는 세부 선을 늘리지 말고 머리, 어깨, 무기, 허리, 발의 다섯 덩어리로 축약한다.

### 2.2 채색과 붓 질감

- 캐릭터: 기본색 + 큰 그림자 + 핵심 접촉 암부의 **2.5단계 셀 채색**. 피부와 넓은 천에만 한 단계의 부드러운 전이를 허용한다.
- 배경: 큰 명도 덩어리를 먼저 나누고, 표면의 10~20%에만 마른 붓·종이결을 얹는다. 사진 같은 미세 자갈·풀·노이즈를 전면에 균일하게 깔지 않는다.
- 하이라이트는 금속, 젖은 물가, 눈동자, 활의 마찰면처럼 재질을 설명하는 곳에만 제한한다.
- 광택 있는 3D 렌더, 사진 합성 같은 표면, 에어브러시만으로 만든 무경계 그라데이션은 금지한다.

### 2.3 색상과 명도

- 기본 팔레트는 먹청, 회청, 흙갈색, 탁한 쪽빛, 적갈색이며 따뜻한 호박·적동색을 서사적 강조로 사용한다.
- 전체 채도는 중저채도, 주목 지점만 주변 대비 15~25% 높은 채도를 허용한다.
- 캐릭터의 얼굴·손·진영색은 배경보다 한 단계 밝거나 채도가 높아야 한다.
- 플레이 공간의 배경 명도 범위는 대체로 18~65%로 제한하고 캐릭터 핵심 실루엣과 최소 20%의 국소 명도 차를 확보한다.
- 검정 압착을 피한다. 가장 어두운 암부에도 남청 또는 적갈색 기운을 남긴다.

### 2.4 조명과 림라이트

- 공통 주광은 장면의 시간대에 맞는 단일 방향광으로 읽히게 한다.
- 따뜻한 크림·호박 림라이트는 화면 주인공, 서사 전환, 새벽과 불빛 장면에서만 사용하며 가장자리 전체가 아니라 광원 반대 실루엣의 15~30%에 선택적으로 둔다.
- 타이틀의 새벽빛도 실제 태양광 범위 안에서 표현하고 광륜·마법 입자·신성 문양으로 초자연을 확정하지 않는다.

### 2.5 비례와 형태

- 원화 인체는 7.5~8등신의 길고 민첩한 비례를 기본으로 한다. 머리와 손은 감정·행동이 읽히도록 현실보다 3~7% 강조할 수 있다.
- 2D 필드 캐릭터는 화면 가독성을 위해 원화보다 머리 8~12%, 손·무기 10~15% 확대하고 천 조각 수는 절반 이하로 줄인다.
- 왕족과 무장은 장식의 양보다 어깨선, 자세, 재료 품질과 색 면적으로 지위를 구분한다.

### 2.6 카메라와 화면 구성

- HD-2D 필드 기준은 직교 카메라, 수직 기준 약 45~50도 내려다보기다. 캐릭터가 바닥에 눕지 않고 상체와 얼굴 방향이 읽혀야 한다.
- 타이틀과 컷신 일러스트는 넓은 여백과 비대칭 구도를 쓰되 인물·로고·메뉴가 놓일 명도 안정 구역을 먼저 확보한다.
- 과도한 광각 원근, 사진식 심도 흐림, 극단적인 시네마틱 크롭은 플레이 화면에서 사용하지 않는다.

## 3. 픽셀·비픽셀 정책

- 프로젝트는 **비픽셀 2D 카툰 스프라이트 + 3D/타일 기반 HD-2D**를 공식 정책으로 한다.
- 픽셀아트처럼 보이게 하는 저해상도 계단선, 최근접 보간, 서로 다른 픽셀 밀도 혼합은 사용하지 않는다.
- 캐릭터 프레임은 384×384 투명 PNG를 유지하되 실제 유효 신장은 모든 방향에서 동일하게 맞춘다. 원본은 고해상도 레이어 파일로 보존하고 게임용은 선명한 축소본으로 출력한다.
- 타일 256×256은 월드 셀 규격이지 회화 밀도 규격이 아니다. 한 셀에 미세 자갈과 풀을 가득 채우지 말고 2~4개의 큰 형태군으로 읽히게 한다.
- 화면 확대·축소는 이방성/선형 필터를 사용하고 100%, 75%, 50% 표시에서 선과 얼굴이 무너지지 않는지 검사한다.

## 4. 원화에서 2D 스프라이트로 변환하는 규칙

1. 원화 승인 전에 정면·측면·후면, 활과 화살통의 실제 결합 위치, 옷자락 길이를 고정한다.
2. 원화의 실루엣을 머리, 몸통, 팔, 하체, 무기, 천의 6개 덩어리로 분해한다.
3. 자수, 작은 끈, 표면 스크래치의 60~80%를 제거하고 진영색 면과 접지 형태를 우선한다.
4. 모든 프레임에서 키, 머리 크기, 활 길이, 화살통 위치, 허리선과 접지점을 고정한다.
5. 보행은 프레임을 복제해 위치만 흔드는 방식이 아니라 접촉-하강-통과-상승 포즈를 새로 그린다.
6. 좌우 반전으로 끝내지 않는다. 활손, 시위손, 화살통과 허리칼의 비대칭을 방향별로 다시 배치한다.
7. 외곽선은 원화보다 20~35% 굵게, 내부선은 40~60% 줄인다. 따뜻한 림라이트는 전투 핵심 프레임에만 1~2 px 유색 면으로 단순화한다.
8. 생성형 원화의 손가락, 활 구조, 끈 연결, 복식 여밈을 사람이 교정한 뒤 프레임 제작에 들어간다.

## 5. 배경 타일에서 HD-2D 화면으로 변환하는 규칙

- 현재 타일의 사진적 미세 질감은 40~60% 줄이고, 풀·흙·물·바위가 큰 색면과 선택적 붓선으로 읽히도록 재도장한다.
- 타일 경계는 반복 배치에서 사라져야 하며 한 셀 내부의 강한 특징은 4×4 영역당 1~3개만 둔다.
- 나무, 바위, 건축물은 캐릭터 외곽선보다 가늘고 저채도인 유색선을 사용한다. 캐릭터 발 주변 0.5~1.0 월드 셀은 시각 잡음을 낮춘다.
- 3D 조명은 텍스처에 이미 그려진 명암과 충돌하지 않게 약한 방향광과 넓은 그림자를 쓴다. 타일에 강한 고정광을 그려 넣지 않는다.
- 물은 사실적인 반사보다 3단 색면, 길게 끊긴 붓결, 제한적인 하이라이트로 표현한다.
- 배경 안개는 공간 분리용이며 캐릭터 전신 대비를 떨어뜨리면 안 된다. 전경 100%, 중경 75~85%, 후경 45~65% 대비를 기준으로 한다.

## 6. 캐릭터·배경 스케일과 대비 기준

- 1280×720 기준 필드 캐릭터의 화면상 유효 신장은 일반 탐색에서 110~155 px를 목표로 한다. 전투는 145~210 px까지 허용한다.
- 주인공 머리 폭은 필드에서 최소 22 px, 활 두께는 최소 2 px를 확보한다.
- 캐릭터 주변 24~40 px 범위에는 동일한 진영색·명도의 큰 오브젝트를 피한다.
- 배경 디테일의 최고 선명도는 캐릭터 얼굴·무기보다 한 단계 낮아야 한다.
- 그림자는 접지용 짧은 타원 또는 부드러운 3D 그림자를 사용하며 캐릭터 하반신을 먹지 않게 불투명도 20~38%로 제한한다.

## 8. 수정 우선순위

### P0 — 통합 전 반드시 해결

1. 타이틀 키비주얼의 사실주의를 폐기하고 `cartoon_v1`과 같은 거친 카툰 배경으로 재제작한다.
2. 북방 황야의 타일 1세트와 나무·바위 각 1종을 카툰 스타일로 다시 만든 골든 배경 셀을 확정한다.
3. 주몽의 필드 정면·측면 키포즈를 원화 축소가 아닌 스프라이트 전용 도안으로 다시 그리고 맵 위에서 스케일·대비를 확정한다.

### P1 — 수직 슬라이스 제작 전 해결

1. `main_cast_v1` 전원의 선, 명암 단계, 배경 효과를 기준 원화에 맞춘다.
2. 캐릭터 방향별 장비 비대칭과 보행 접지를 교정한다.
3. 프롤로그 강가와 훈련장 장면에서 조명·안개·그림자 강도를 통합한다.
4. 타이틀 로고와 메뉴도 같은 먹선·적동 강조 체계를 사용하되 텍스트 가독성은 질감보다 우선한다.

### P2 — 양산 단계에서 해결

1. 지역별 보조 팔레트와 계절·날씨 변형을 추가한다.
2. 초상화 6종 표정과 전투 이펙트의 선 굵기 규격을 확장한다.
3. Steam 캡슐, 로딩 화면, 아이콘 파생 규격을 만든다.

## 9. 승인용 골든 샘플

대량 제작 전에 아래 한 묶음을 사용자와 PM이 화면으로 승인해야 한다.

1. **캐릭터 원화:** 17세 주몽 전신 1장. 정면에 가까운 3/4 자세, 중립광, 배경 없음. 선·2.5단 명암·재료·적갈색 기준을 포함한다.
2. **필드 스프라이트:** 주몽 정면 대기 1프레임, 좌측 보행 4프레임, 활 준비 1프레임. 384×384 원본과 실제 화면 크기 합성본을 함께 제출한다.
3. **배경 셀:** 풀, 흙, 얕은 물, 물가 전환 각 1셀과 나무·바위 각 1종. 반복 4×4 미리보기를 포함한다.
4. **통합 화면:** `PrologueRiverbankMap` 구도의 1280×720 화면 한 장. 주몽 대신 어부 스케일 더미를 포함하고 전경·중경·후경, 안개와 새벽빛을 보여준다.
5. **타이틀:** 같은 강가 구도를 비픽셀 카툰 배경화로 만든 1920×1080 이미지 한 장. 정확한 Unicode 임시 제목과 메뉴를 합성해 가독성을 확인한다.

승인 체크:

- 축소했을 때 머리·어깨·무기·진영색이 즉시 구분되는가?
- 원화, 스프라이트, 배경, 타이틀이 같은 선과 안료로 만들어진 것처럼 보이는가?
- 캐릭터가 배경보다 한 단계 선명하고 밝게 읽히는가?
- 초자연 요소를 사실로 확정하거나 타 작품을 연상시키는 상징·복식이 없는가?
- 반복 타일 경계, 생성형 인체·무기 오류, 한글 글자 오류가 없는가?

골든 샘플이 승인되기 전에는 전체 캐릭터, 전체 타일, 정식 타이틀의 대량 생산을 시작하지 않는다.

## 10. 파일 소유와 검수 절차

- 아트 디렉션·팔레트·골든 샘플 승인: PM/통합 아트 디렉터
- 원화와 타이틀 배경: 일러스트 담당
- 필드·전투 스프라이트: 2D 캐릭터 담당
- 타일·오브젝트와 월드 합성: 맵 담당
- UI 치수·상태·접근성: UI/UX 담당
- Godot 임포트·씬·조명: 구현 담당
- 화면·로그·종료 코드 판정: 검증 담당

모든 담당자는 골든 샘플을 입력 기준으로 사용하고 동일 기준 파일을 동시에 수정하지 않는다. AI 생성 애셋은 도구, 날짜, 목적, 수정 여부, 사용 위치와 저작권 검토를 기록하며 사람이 구조와 상업 사용 적합성을 검수한다.
