# 분대 교전 컷인 2D 블록아웃 v1

## 목적

전술 맵에서 계산이 끝난 뒤 지휘관과 병사 다수의 충돌을 1~3초 안에 보여 주는 전체 2D 기준선이다. 이 장면은 계산, 충돌, 사망 판정, 상태 전이를 소유하지 않고 전달받은 payload를 표현한다.

## 화면 정본

- 양측 모두 지휘관 1명과 병사 최대 8명을 별개의 실루엣으로 그린다.
- 약 3등신 SD 도트 캐릭터를 목표로 하며 블록아웃은 사각 픽셀 덩어리와 제한 팔레트로 표현한다.
- 바닥은 45~50도 쿼터뷰를 읽을 수 있는 마름모 무대와 대각 격자로 구성한다.
- 궁병은 활의 곡선과 후열 일제사격, 창병은 긴 창과 전열 압박으로 구분한다.
- 손실은 사망 연출이 아니라 병사 수 감소와 뒤로 물러나는 동작으로 표현한다.
- 상단의 지휘관 마름모와 병사 8칸 표시는 언어에 의존하지 않고 전후 병력을 보조한다.

## 시간 구성

| 구간 | 기준 시간 | 화면 동작 |
|---|---:|---|
| 공격 준비 | 0.42초 | 공격 측이 호흡을 모으며 반보 물러난다. |
| 충돌·사격 | 0.56초 | 궁병은 순차 화살 6발, 근접병은 전열 충격선을 표시한다. |
| 피해·후퇴 | 0.44초 | payload의 전후 병력 차이만큼 병사가 빠지고 방어 측이 후퇴한다. |

기본 총 길이는 1.42초다. `playback_speed`는 접근성·고속 진행과 자동 검증을 위한 표현 속도이며 전투 결과에는 영향을 주지 않는다.

## 런타임 계약

- 장면: `res://scenes/combat/presentation/squad_clash_cutin_2d.tscn`
- API: `play_clash(payload)`, `skip()`
- 완료 신호: `presentation_finished(payload)`
- 큐: 재생 중 새 payload가 들어오면 순서대로 보관하고, 현재 연출 완료 또는 건너뛰기 뒤 다음 연출을 시작한다.
- 양측 기본 배치는 공격자가 왼쪽이다. 적 공격처럼 방향을 바꿔야 할 때 선택 필드 `attacker_side = "right"`를 전달한다.
- 선택 필드 `attacker_color`, `defender_color`, `playback_speed`는 표현만 바꾸며 공통 계산 payload를 훼손하지 않는다.

## 해상도·입력

- 루트 `Control`은 전체 화면 앵커를 사용해 1280×720과 1280×800에서 중앙 무대와 상단 병력 표시를 안전 영역 안에 둔다.
- 우상단 `▶▶` 버튼은 마우스·키보드·게임패드 포커스를 지원한다.
- `ui_cancel`과 `game_cancel`도 현재 컷인을 즉시 완료한다.
- 건너뛰기와 정상 완료는 입력받은 원래 payload를 그대로 완료 신호로 방출하므로 계산과 승패가 바뀌지 않는다. 화면상 병사는 최대 8명만 표시하되 데이터의 6~12명 값은 수정하지 않는다.

## 다음 아트 패스

플레이스홀더 승인 뒤 3등신 지휘관·궁병·창병의 컷인 전용 측면 대각 2방향을 먼저 제작한다. 필요한 애니메이션은 준비, 공격, 피격, 비살상 후퇴이며, 접지 그림자와 타격 VFX는 캐릭터와 같은 2D 캔버스에서 제작한다. 특정 게임의 대형·UI·타격 구도를 복제하지 않는다.
