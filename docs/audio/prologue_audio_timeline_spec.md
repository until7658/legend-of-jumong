# 오프닝 14컷 연출·오디오 명세

## 원칙

- 총 길이 264초. 컷 시간은 `prologue_timeline_events.json`의 로컬 초를 정본으로 한다.
- 내레이션에는 초상화를 표시하지 않는다. 대사에는 `speaker_id`와 전용 `expression_id`가 반드시 존재해야 하며 누락 시 인물 기본 표정으로 폴백한다.
- 유화는 프롤로그에서 말하지 않는다. 신하와 대소를 악역 스팅어로 규정하지 않고, 빛과 음악으로 초자연을 사실로 확정하지 않는다.
- 스킵은 각 컷의 `skip_endpoint.state`를 즉시 적용하고 0.15초 크로스페이드한다. 루프·덕킹·초상화·카메라가 남지 않아야 한다.

## 동기 구조

1. 컷 1: 음악 없이 강·바람·목재만으로 현실을 연다.
2. 컷 2~3: 낮은 현의 불완전한 간격으로 발견의 긴장을 만들고, 호흡 확인 뒤 `human_trace` 2음 씨앗을 둔다.
3. 컷 4~6: 씨앗이 낮은 현·목관의 짧은 인간적 동기로 확장된다.
4. 컷 7~9: 같은 음정을 낮은 불협과 목재 룸톤으로 바꿔 궁궐의 경계를 표현한다.
5. 컷 10~12: 새·잔잔한 바람과 낮은 목관으로 긴장을 풀고 각성 동기로 전환한다.
6. 컷 13~14: 앞선 2~3음이 처음 완전한 타이틀 테마가 된다. 합창·마법 벨·영웅 팡파르는 사용하지 않는다.

## 믹스와 납품

- Dialogue, Music, Ambience, SFX 버스를 분리한다. 대사 중 음악 -9dB, 환경음은 -4dB 추가 덕킹을 시작 120ms/복귀 350ms로 적용한다.
- 강물·배 삐걱임·노/그물·발걸음·젖은 옷·호흡·문·실내·바람을 하나의 합성 파일로 묶지 않는다. 화면 동작과 개별 동기화하고 접근성/믹스 조정을 가능하게 한다.
- 루프는 48kHz/24bit 마스터에서 최소 2회 반복 청취하고 제로크로싱, 위상, 모노 호환, 클릭을 검사한다.
- 모든 오디오는 누락 시 무음으로 장면을 계속한다. 오디오 로드가 내러티브 진행을 블로킹해서는 안 된다.

## 타이틀 WAV 검토

`assets/audio/music/title_river_at_dawn.wav`는 파일 존재(14,817,644 bytes)와 기존 JSON의 `unverified_placeholder` 분류만 확인했다. 현재 환경에는 `ffprobe`가 없어 헤더/음압/청취 검증을 완료하지 못했다. 따라서 컷 13~14의 33초 편집에 맞는지, 주제 동기인지, 상업 이용 가능한지 확정할 수 없다. 승인 전 샘플레이트·비트·채널·길이·피크·LUFS·클릭·출처·라이선스·AI 생성 기록을 검사하고, 실패 시 `music.theme.full` 신규 제작으로 교체한다.

## 초상화 추가 요구

- P0: `fisherman/breath_check_concern`, `yuhwa/unconscious_wet`, `yuhwa/awakening_disoriented`.
- 기존 사실주의/회화형 초상화는 강한 피부 묘사와 림라이트 때문에 카툰형 02의 거친 외곽선·2.5단 셀 명암과 불일치한다. 전용 컷신 초상화는 카툰형 02로 재제작하며 기존 파일은 임시 폴백으로만 둔다.

## 검증

- JSON 두 파일을 UTF-8로 파싱한다.
- 모든 컷의 duration, skip endpoint, 카메라, 맵 상태, 배우 상태를 검사한다.
- dialogue 이벤트마다 portrait ID가 있고 narration 이벤트에는 portrait가 없는지 검사한다.
- 음악 exclusive 전환, 덕킹 복귀, 루프 종료, 컷14 이후 무음 상태를 런타임에서 확인한다.
- 실제 오디오가 없으므로 현재 판정은 설계 계약이며 청취 검증 불가다.
