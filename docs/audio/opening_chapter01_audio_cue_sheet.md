# 오프닝·제1장 사운드 큐시트 v1

## 오디오 방향

사운드는 영웅의 운명을 미리 선언하지 않고 **물, 바람, 목재, 말, 활, 북**이라는 현실적 소리에서 신화의 여운을 만든다. 음악은 저음 현·목관·절제된 북과 숨결 같은 지속음을 중심으로 하며 현대적 신스 영웅 테마, 과장된 합창, 마법 반짝임은 사용하지 않는다.

- 프롤로그: 차갑고 가까운 강 소리에서 시작해 사람의 구조와 판단을 거쳐, 컷 13에서만 주제가 온전한 선율을 이룬다.
- 제1장: 17세 주몽의 민첩함과 긴장을 표현한다. 승리 팡파르보다 활·바람·관중의 억눌린 반응을 우선한다.
- 대소: 노골적 악역 모티프 금지. 낮은 현의 짧은 불협과 공간의 침묵으로 불안을 만든다.
- 유화: 천, 약재, 손 움직임과 낮은 목관으로 보호와 결단을 표현한다. 신성한 여성 합창은 금지한다.
- 비살상 전투: 타격의 잔혹함보다 밧줄·나뭇가지·발소리·호흡·포위망 이탈을 강조한다.

## 런타임 계약

정본 데이터는 `data/audio/opening_chapter01_cues.json`이다.

- 호출자는 `event_id`만 발행한다. 오디오 시스템은 `asset_id`, `loop`, `fade_in_ms`, `fade_out_ms`, `fallback`을 해석한다.
- `trigger.type`은 `narrative_event`, `map_enter`, `map_exit`, `gameplay_signal`, `timeline_time`, `ui_state` 중 하나다.
- `trigger.ref`는 내러티브/맵 담당의 공개 ID다. 확정 전 임시 ID에는 `contract_status: provisional`을 둔다.
- 현재 내러티브 공개 계약은 `sequence_started(sequence_id)`, `entry_changed(sequence_id, index, entry)`, `sequence_finished(sequence_id)`이며 정본 ID는 `prologue_river_woman_v1`, `chapter_01_nameless_prince_v1`이다. 컷 단위 marker는 통합 어댑터가 `entry_changed`에서 변환한다.
- 현재 맵 공개 계약은 `map_ready`, `scene_started`, `scene_finished`이며 정본 ID는 `prologue_riverbank`, `chapter_01_training_ground`이다.
- `exclusive_group`이 같은 루프는 새 이벤트가 시작될 때 기존 루프를 `fade_out_ms`로 종료한다.
- 파일이 없거나 로드에 실패하면 `fallback.type`에 따라 무음, 다른 이벤트, 엔진 생성 톤/노이즈 중 하나를 사용한다. 프로덕션 빌드에서 엔진 생성 톤은 접근성·기능 검사용 플레이스홀더일 뿐이다.
- 대사 재생 중 음악은 `dialogue_duck_db`만큼 낮추고 환경음은 핵심 대사 가독성을 해치지 않게 별도 버스로 제어한다.

## 제작 우선순위와 납품

### P0 — 기능 수직 슬라이스

1. 강·실내·훈련장·능선·왕성 경보의 환경 루프 5종
2. 활 당김·릴리스·화살 통과·표적/매듭·목문·성문·발소리·말·북 핵심 SFX
3. 프롤로그 인간 흔적 모티프, 컷 13 주제, 유화 결별, 사냥터 긴장 음악 스케치
4. 모든 파일 누락 시 안전한 무음 fallback과 자막/시각 피드백

### 파일 규격

- 마스터: WAV, 48kHz, 24bit. 루프는 클릭 없는 시작/종료 지점과 2회 반복 검증본을 제공한다.
- 게임 임포트 후보: 음악·긴 환경음은 Ogg Vorbis, 짧은 UI/SFX는 WAV. 최종 Godot 임포트는 구현 담당이 확정한다.
- 권장 경로: `assets/audio/{music,ambience,sfx,ui}/opening_chapter01/`.
- 파일명은 JSON `asset_id`의 마지막 요소와 일치시키고 영문 소문자·숫자·밑줄만 사용한다.
- 각 큐는 시작/종료 무음, 피크, 루프 클릭, 대사 덕킹, 모노 호환성을 검수한다.

## 검증 체크리스트

- 음원이 없어도 장면 진행과 대사 타이밍이 멈추지 않는가?
- 프롤로그에 마법·신격화를 확정하는 합창·벨·반짝임이 없는가?
- 대소와 신하가 단순 악역 음악으로 규정되지 않는가?
- 활 시위, 화살, 표적의 효과음이 화면 동작과 프레임 단위로 맞는가?
- 대사 중 음악 덕킹과 자막 타이밍이 일치하는가?
- 루프 전환과 맵 이동에서 클릭·중첩·갑작스러운 음압 상승이 없는가?
- 마스터 음소거·음악·효과음 슬라이더와 모션/접근성 옵션에 반응하는가?

## 의존성과 위험

- 내러티브 이벤트와 맵 ID가 확정되어야 `trigger.ref`를 동결할 수 있다.
- 현재 실제 승인 음원은 없다. 모든 신규 cue는 무음 fallback을 기본으로 해야 한다.
- 장면 길이와 컷 편집이 바뀌면 음악의 마디보다 `timeline_time`과 대사 종료 신호를 우선 재조정한다.
- 외부·생성형 음원을 도입하면 상업 이용, 학습 데이터 고지, 저작권·실연권·샘플 권리를 별도 검토한다.
