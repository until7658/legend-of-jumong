## M4 오전 추가 보고 · SD 도트 쿼터뷰 분대전

### 사용자 확정 방향

- 인게임 캐릭터는 2~4등신, 목표 약 3등신의 도트 캐릭터로 제작한다.
- 필드와 전술 맵은 45~50도 하향 쿼터뷰를 유지한다.
- 전투는 지휘관이 다수 병사를 편성·지휘하는 분대형 턴제 전술 전투로 전환한다.
- 분대는 총 4명이 아니다. 데이터상 지휘관 1명+병사 6~12명 가변이며 M4는 지휘관 1명+병사 8명을 표시한다.
- 전술 맵, 다수 병사 교전 컷인, 궁술대회·대화·결투 같은 개인 컷신을 서로 다른 연출 레이어로 구성한다.

### 에이전트 협의 결과

- production/game direction: 특정 전술 RPG의 UI·수치·구도는 복제하지 않고 `군령`, `대열`, `기세`를 프로젝트 고유 규칙으로 사용한다.
- combat: 궁병은 2칸 이상에서 창병에 유리하고 창병은 인접한 궁병에 유리한 위치 상성을 M4에서 먼저 검증한다.
- character/2d-art: 전술 맵은 8인 축약 대형, 교전 컷인은 양측 16명 규모 블록아웃, 개인 컷신은 주몽의 자세·호흡·활·화살 궤적 중심을 추천한다.
- ui/core: 1280×720 기준 viewport 정수 배율, Nearest 필터, 픽셀 스냅, 키보드·게임패드 공통 행동을 적용한다.
- audio/Steam: 전술 명령음·교전 충격음·서사 감정음을 분리하고 교전 컷인은 짧고 건너뛸 수 있게 설계한다.

### 구현

- `SquadBattleController` 명시적 상태머신: 선택→이동→군령→해결→적 턴→라운드 종료→승패
- 주몽+수련 궁병 8명 대 훈련대장+수련 창병 8명
- 이동, 기본 사격, 대열 방어, 턴 종료, 거리/인접 상성, 병사 기세와 지휘관 기세, 비살상 제압
- 타이틀→약 90초 프롤로그→분대 훈련전→결과→`m4_training_complete` 체크포인트 저장 흐름
- 런타임 입력 프로필 8개 행동과 버전 1 JSON 세이브 서비스
- 약 3등신 주몽 남향 도트 후보를 생성하고 투명화했으며 AI 애셋 대장에 `approval_candidate`로 기록
- 이전 6등신 비픽셀 신규 후보는 방향 폐기로 프로젝트에서 제거
