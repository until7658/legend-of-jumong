# 타이틀 화면 UX·접근성 승인 기준

## 1. 문서 목적과 적용 범위

이 문서는 `scenes/ui/title_screen.tscn`, `scripts/ui/title_screen.gd`, `shaders/title_river_background.gdshader` 및 타이틀에서 다음 장면으로 이어지는 부트스트랩 동작의 승인 기준을 정의한다. 기능이 파싱된다는 사실과 사용자가 화면을 보고 조작할 수 있다는 사실을 분리해 검증한다.

- 담당 영역: `ui-systems`, `asset`, `verifier`
- 기준 해상도: 1280×720
- 대상 입력: 키보드, 마우스, 게임패드
- 대상 메뉴: 새 게임, 이어하기, 설정, 종료
- 현재 프로토타입의 다음 장면은 HD-2D 시험장이지만 정식 승인 대상은 `새 게임 → 오프닝 「강에서 온 여인」` 전환이다.
- 검증 중 코드, 씬, 리소스, 애셋은 수정하지 않는다.

## 2. 승인 판정 원칙

- **통과:** 모든 필수 기준이 화면·입력·로그 증거로 확인되고 차단 오류가 없다.
- **부분 통과:** 일부 기준만 확인되거나 정식 오프닝, 오디오, 접근성 옵션 등 미구현 항목이 남는다.
- **실패:** 진행 차단, 조작 불능, 잘림, 포커스 상실, 잘못된 장면 전환 또는 오류가 재현된다.
- **검증 불가:** 필요한 장치, GUI 환경, 빌드 또는 증거를 확보하지 못했다.

종료 코드 0과 파싱 성공만으로 화면·입력·전환을 통과시키지 않는다. 각 기준에는 재현 절차와 관찰 가능한 증거가 있어야 한다.

## 3. 기능 승인 기준

### 3.1 최초 진입

- 게임 실행 시 타이틀 화면이 최초의 상호작용 화면으로 나타난다.
- 월드와 HUD는 타이틀 뒤에 노출되거나 입력을 받지 않는다.
- 최초 포커스는 활성 메뉴인 `새 게임`에 있다.
- 타이틀, 부제, 메뉴, 버전 표기가 한 프레임 안에서 겹치거나 잘리지 않는다.
- 초기화 과정에서 오류, 누락 리소스, 셰이더 컴파일 오류가 발생하지 않는다.

### 3.2 새 게임

- `새 게임`을 한 번 선택하면 중복 입력을 차단하고 전환은 한 번만 실행된다.
- 선택 직후 메뉴 입력이 잠기고, 전환 중 포커스나 클릭이 뒤 장면으로 전달되지 않는다.
- 정식 목표에서는 페이드 뒤 오프닝 「강에서 온 여인」의 첫 장면으로 이동한다.
- 오프닝 진입 후 타이틀 화면의 오디오, 입력, 애니메이션과 노드가 남지 않는다.
- 전환 실패 시 검은 화면이나 무응답 상태에 머물지 않으며 오류가 로그에 식별 가능하게 남는다.

### 3.3 이어하기·설정

- 미구현 상태에서는 두 메뉴가 선택·확정되지 않는다.
- 비활성 상태는 색만으로 구분하지 않고 명도, 테두리/아이콘 또는 보조 문구를 함께 사용한다.
- 포인터를 올리거나 포커스 탐색이 인접할 때 다음 안내 중 하나가 노출된다.
  - `이어하기: 저장 데이터가 없습니다.`
  - `설정: 현재 프로토타입에서는 사용할 수 없습니다.`
- 안내는 최소 2초 유지되거나 사용자가 초점을 옮길 때까지 표시되고, 스크린 리더를 도입할 경우 상태 텍스트로도 전달한다.
- 비활성 메뉴 때문에 포커스 이동이 끊기거나 갇히지 않는다.

### 3.4 종료

- `종료` 확정 시 프로세스가 정상 종료된다.
- `ui_cancel` 입력은 실수로 즉시 종료시키지 않도록 확인 대화상자 또는 명시적 2단계 확인을 제공하는 것을 정식 승인 조건으로 한다.
- 확인 대화상자의 기본 포커스는 안전한 선택인 `취소`이며, 취소하면 이전 메뉴와 포커스로 돌아간다.

## 4. 화면·레이아웃 승인 기준

- 제목이 최우선, 부제가 두 번째, 메뉴가 세 번째 정보 위계를 갖는다.
- 밝은 새벽빛·안개·수면 변화 위에서도 모든 텍스트의 가독성이 유지된다.
- 제목과 메뉴는 화면 가장자리에서 최소 5% 안전 영역을 확보한다.
- 버전 표기는 최소 글자 크기와 대비를 충족하며 창 크기가 작아져도 잘리지 않는다.
- 배경은 늘어나거나 압축되어 수평선이 비정상적으로 휘지 않는다. 종횡비가 달라질 때 중앙 크롭 또는 별도 레이아웃 정책을 사용한다.
- 메뉴 상태는 기본, 포인터 오버, 키보드/게임패드 포커스, 누름, 비활성의 다섯 상태가 시각적으로 구별된다.
- 포커스 표시는 색상 변화만 사용하지 않는다. 외곽선, 밑줄, 마커 또는 크기 변화 중 하나를 병행한다.
- 포커스 외곽선은 배경과 인접 상태 대비가 최소 3:1이어야 하며, 텍스트는 일반 크기 기준 최소 4.5:1을 목표로 한다.

## 5. 입력 승인 기준과 테스트 매트릭스

| 입력 장치 | 이동/포커스 | 확정 | 취소 | 포인터 상태 | 필수 증거 |
|---|---|---|---|---|---|
| 키보드 | 위/아래 방향키로 활성 메뉴만 순환, 포커스 유실 없음 | Enter 또는 Space로 1회 실행 | Esc로 종료 확인창 열기, 다시 Esc로 닫기 | 해당 없음 | 최초 포커스, 전체 순환, 새 게임 전환 영상/연속 캡처 |
| 마우스 | 활성 버튼 호버 시 포커스/호버 상태 명확 | 좌클릭 1회로 1회 실행, 더블클릭 중복 방지 | 종료 버튼은 확인창 경유 | 비활성 메뉴 커서와 사유 안내 확인 | 각 버튼 상태 스크린샷, 전환 로그 |
| 게임패드 | D-pad와 왼쪽 스틱으로 활성 메뉴만 순환, 데드존 오작동 없음 | 표준 South/A/Cross 버튼 | East/B/Circle 버튼으로 종료 확인창/취소 | 해당 없음 | 사용 패드 모델, 입력 매핑, 순환과 전환 영상 |

추가 기준:

- 입력 장치가 바뀌어도 포커스가 사라지지 않는다.
- 비활성 버튼을 건너뛴 뒤 위·아래 이동 순서가 예측 가능하다.
- 버튼을 누른 채 유지해도 새 게임과 종료 신호가 반복 발생하지 않는다.
- 게임패드 연결·해제 후 키보드와 마우스 조작이 계속 가능하다.

## 6. 접근성 승인 기준

### 6.1 글자와 한글 글꼴

- 한글 전체 음절, 문장부호, 숫자, 영문 버전 표기에 누락 글리프가 없다.
- 프로젝트에 승인된 한글 UI 글꼴을 포함하고 라이선스와 사용 위치를 기록한다.
- 기본 글꼴 실패 시 한글을 지원하는 명시적 fallback 체인을 제공한다. 운영체제 글꼴 존재만 가정하지 않는다.
- 1280×720 기준 메뉴 본문은 실측 18px 이상, 보조 문구는 16px 이상을 목표로 하며 작은 화면에서도 읽을 수 있어야 한다.
- 글자 확대 125% 또는 OS 배율 변화에서도 제목·부제·메뉴가 겹치거나 잘리지 않는다.

### 6.2 대비와 상태 전달

- 텍스트 대비는 일반 텍스트 4.5:1, 큰 텍스트 3:1 이상을 목표로 실측한다.
- 활성/비활성, 포커스/비포커스 상태를 색 하나에만 의존하지 않는다.
- 움직이는 안개와 반사 위에서 최저 대비 지점을 캡처해 측정한다.
- 비활성 사유는 화면 텍스트로 제공하고, 단순히 흐리게 표시하는 것으로 끝내지 않는다.

### 6.3 모션 감소

- `모션 감소`가 켜지면 안개·수면 반사와 기타 연속 배경 움직임을 정지 또는 현저히 감속한다.
- 새 게임 페이드는 번쩍임 없이 0.1~0.3초의 단순 전환 또는 즉시 전환으로 대체할 수 있다.
- 어느 모드에서도 초당 3회 이상 번쩍이는 효과가 없어야 한다.
- 모션 설정은 타이틀 진입 전에 저장값을 읽어 적용하며, 설정 화면이 미구현인 프로토타입에서는 테스트용 기본값과 제한을 명시한다.

## 7. 모션·오디오 승인 기준

### 7.1 모션

- 기본 배경 움직임은 메뉴 탐색을 방해하지 않고 멀미를 유발할 정도의 시차 이동이 없다.
- 최초 진입, 포커스 이동, 확정, 전환 애니메이션은 각각 목적이 구분된다.
- 전환 중 입력 잠금이 시각적으로 전달된다.
- 프레임 속도가 낮아져도 페이드 완료 이벤트와 장면 전환이 누락되지 않는다.

### 7.2 오디오

- 타이틀 환경음, 음악, 포커스 이동음, 확정음, 취소음, 비활성음의 역할과 음량이 분리된다.
- 포커스 이동음은 연속 입력 시 과도하게 중첩되지 않는다.
- 비활성 메뉴를 시도하면 확정음과 다른 짧은 안내음이 재생된다.
- 새 게임 선택 후 타이틀 음악·환경음이 중복 재생되지 않고 오프닝의 강물·바람 소리로 자연스럽게 이어진다.
- 마스터 음소거와 저장된 음량 설정을 시작 시점부터 적용한다.
- 오디오 장치가 없거나 변경되어도 오류로 진행이 중단되지 않는다.

## 8. 해상도·창 모드 승인 기준

각 환경에서 전체 화면과 창 모드를 확인한다. 운영체제 DPI는 최소 100%와 125%를 포함한다.

| 화면군 | 대표 해상도 | 승인 기준 |
|---|---:|---|
| 16:9 기준 | 1280×720, 1920×1080, 2560×1440 | 기준 구도 유지, 텍스트·메뉴·버전 잘림 없음 |
| 16:10 | 1280×800, 1920×1200 | 추가 세로 공간이 있어도 콘텐츠가 과도하게 벌어지지 않음 |
| 울트라와이드 | 2560×1080, 3440×1440 | 콘텐츠는 중앙 안전 영역 유지, 배경 가장자리 빈 공간·왜곡 없음 |
| 작은 창 | 960×540 또는 지원 최소 크기 | 최소 글자 크기 유지, 필요하면 전체 UI 스케일 적용 |
| 창 크기 변경 | 최소↔최대 드래그 | 포커스와 입력 유지, 한 프레임 이상 심각한 겹침·깜박임 없음 |

추가 기준:

- 전체 화면 전환 후 포커스가 `새 게임` 또는 이전 선택 메뉴에 남는다.
- 화면 밖 클릭이나 Alt+Tab 복귀 후 게임패드/키보드 포커스가 복구된다.
- 울트라와이드에서 셰이더 수평선과 새벽빛의 의미 있는 구도가 보존된다.

## 9. 새 게임 → 오프닝 전환 승인 기준

- `새 게임` 확정 시 정확히 한 번 전환 신호가 발생한다.
- 타이틀 페이드, 오디오 페이드, 오프닝 로딩의 순서가 명확하다.
- 오프닝 첫 프레임은 검은 화면과 강물·바람 소리 또는 승인된 첫 컷으로 시작한다.
- 전환 뒤 입력 포커스가 폐기된 타이틀 노드를 참조하지 않는다.
- 타이틀 노드와 셰이더가 해제되어 오프닝 성능과 오디오에 영향을 주지 않는다.
- 현재 HD-2D 시험장 전환은 프로토타입 검증에만 인정하며 정식 타이틀 승인에는 인정하지 않는다.

## 10. 오류 로그와 증거 요구사항

### 10.1 필수 실행 기록

- Godot 버전과 렌더러
- 운영체제, 화면 해상도, DPI, 창/전체 화면 상태
- 입력 장치 종류와 게임패드 모델
- 실행 명령과 종료 코드
- Godot 오류·경고 개수 및 주요 메시지
- 새 게임 신호와 오프닝 진입을 식별할 수 있는 로그

파싱 오류, 셰이더 컴파일 오류, 누락 리소스, invalid node reference, 중복 장면 전환, orphan node 관련 오류는 0건이어야 한다. 비차단 경고도 목록과 영향 분석을 남긴다.

### 10.2 필수 스크린샷·영상 증거

- 16:9, 16:10, 울트라와이드, 작은 창 각각의 최초 화면
- `새 게임` 키보드/게임패드 포커스 상태
- 활성·호버·누름·비활성 버튼 상태
- 이어하기·설정의 비활성 사유 안내
- 종료 확인 대화상자와 안전한 기본 포커스
- 모션 감소 전·후 화면 또는 설정값과 정지 상태를 보여 주는 짧은 영상
- 새 게임 확정 직전, 페이드 중, 오프닝 첫 프레임의 연속 증거
- 한글 fallback 검증용 강제 주 글꼴 누락 테스트 화면

증거 파일은 재검증 가능한 경로, 촬영 조건, 날짜를 함께 기록한다. 확보하지 못한 증거는 이유와 함께 `미확인`으로 보고한다.

## 11. 현재 프로토타입 위험 목록

아래 평가는 파일 정적 검토에 근거하며 실제 GUI 조작 결과가 아니다.

### P0 — 정식 승인 차단

- **새 게임이 오프닝이 아닌 HD-2D 시험장으로 이동한다.** `scripts/bootstrap/main.gd`는 타이틀 종료 후 시험장과 HUD를 표시한다. 정식 흐름인 오프닝 「강에서 온 여인」과 불일치한다.
- **취소 입력이 확인 없이 즉시 게임을 종료한다.** `ui_cancel`이 바로 `quit_requested`를 방출해 키보드·게임패드 오입력으로 진행이 종료될 수 있다.

### P1 — 높은 접근성·조작성 위험

- **비활성 메뉴의 사유 안내가 없다.** 이어하기와 설정은 흐리게 비활성화될 뿐 저장 데이터 부재 또는 미구현 상태를 설명하지 않는다.
- **명시적 한글 글꼴과 fallback이 없다.** 씬은 글꼴 크기만 지정하므로 플랫폼별 기본 글꼴과 한글 글리프 지원에 의존한다.
- **모션 감소 경로가 없다.** 셰이더의 `TIME` 기반 수면·안개 움직임을 정지하거나 줄일 설정이 없다.
- **게임패드 흐름이 명시적으로 설계·검증되지 않았다.** 기본 UI 액션에 의존하며 D-pad/스틱/확정/취소 매핑과 장치 교체가 확인되지 않았다.
- **포커스 상태의 비색상 표시가 보장되지 않는다.** 버튼별 focus StyleBox나 마커가 정의되지 않아 적용 테마에 따라 포커스 가시성이 부족할 수 있다.

### P2 — 품질·확장 위험

- **1280×720 외 해상도별 배치 정책이 정의되지 않았다.** 중앙 고정 오프셋과 우하단 버전 표기는 작은 창, 16:10, 울트라와이드에서 실제 확인이 필요하다.
- **대비가 움직이는 배경의 최저 지점에서 측정되지 않았다.** 제목에는 그림자가 있지만 부제·버전·버튼은 배경 변화에 따라 대비가 낮아질 수 있다.
- **타이틀 전용 오디오 UX가 없다.** 선택·확정·비활성·환경음 및 오프닝으로의 오디오 전환이 확인되지 않는다.
- **종료 포커스 이웃이 비활성 설정 버튼을 가리킨다.** 기본 컨테이너 탐색이 비활성 항목을 건너뛸 가능성은 있으나 키보드와 게임패드 순환을 실제 검증해야 한다.
- **버전 표기 13px은 작은 글자로 분류될 가능성이 높다.** DPI와 화면 거리 조건에서 가독성 확인 및 확대 대응이 필요하다.
