# 타이틀 화면 디자인 시안 검토

## 1. 검토 범위와 증거

1280×720 축소 사용을 전제로 다음 시안을 직접 시각 검사했다. 원본 파일은 수정하지 않았다.

- 와이어프레임: `assets/ui/title/title_screen_wireframe.svg` (1280×720)
- 키비주얼 01: `assets/ui/title/keyvisual_dawn_river_rough_01.png` (1920×1080)
- 키비주얼 02: `assets/ui/title/keyvisual_dawn_river_rough_02.png` (1920×1080)
- 로고 러프 시트: `assets/ui/title/title_logo_rough_sheet.png` (1536×1024)
- 검사 방식: `view_image`로 PNG 3종 직접 확인, SVG 원문에서 좌표·크기·색·텍스트 확인
- 검사 한계: 실제 Godot 합성 화면, 1280×720 래스터 출력, 대비 수치 측정, 입력 상태 애니메이션은 아직 확보하지 못했다.

## 2. 와이어프레임 검토

### 장점

- 1280×720의 16:9 캔버스 안에서 로고(300,126, 680×96), 부제(300,232, 680×28), 메뉴(476,356, 328×212)가 중앙 축에 정렬돼 정보 위계가 명확하다.
- 외곽 안전영역은 좌우 48px, 상하 40px이다. 로고와 메뉴는 이 경계에서 충분히 떨어져 있고, 16:9 기준 잘림 위험이 낮다.
- 중앙 로컬 스크림(276,104, 728×486)이 로고부터 메뉴까지 한 덩어리로 보호한다. 키비주얼의 명도 변화에 대응할 수 있는 현실적인 구조다.
- `새 게임` 포커스는 밝은 외곽선, 좌측 적동색 표식, 배경 명도 변화로 표현되어 색상 하나에만 의존하지 않는다.
- 비활성 메뉴와 함께 하단에 `저장된 여정이 없습니다.`를 배치해 단순 회색 처리보다 상태 설명이 낫다.

### 보완점

- 좌우 48px은 캔버스 폭의 3.75%로, 일반적인 5% 안전 여백보다 좁다. PC 실행 화면에서는 허용 가능하지만 오버스캔·캡처·플랫폼 확장을 고려하면 핵심 텍스트는 좌우 64px 이상을 권장한다.
- 하단 도움말은 `이어하기` 비활성 사유만 전달한다. `설정`이 비활성인 이유는 포커스 인접, 포인터 호버 또는 별도 상태 문구로 구분해야 한다.
- 메뉴의 상태 설계는 포커스 1종만 구체화됐다. 기본, 호버, 누름, 비활성, 전환 잠금 상태를 같은 시트에서 비교해야 한다.
- 16:9 전용 좌표이므로 16:10, 울트라와이드, 작은 창에서 중앙 UI의 최대 폭과 하단 기준선 정책이 별도로 필요하다.
- 13px 버전 표기는 1280×720 축소 및 높은 DPI에서 작다. 최종 화면에서는 최소 16px 상당 또는 UI 배율 적용을 권장한다.

## 3. 키비주얼 비교

### 3.1 키비주얼 01

관찰:

- 어두운 좌측 산과 전경 암석, 중앙의 넓은 강, 우측의 강한 새벽빛이 대비되는 극적인 구도다.
- 작은 배가 화면 중앙보다 약간 오른쪽에 있어 `강에서 시작된 이야기`를 직접 암시한다.
- 좌우 전경 식생과 암석이 프레임 역할을 하며 깊이감이 강하다.
- 중앙 상단은 어두운 구름, 중앙 하단은 물결 세부가 이어진다. UI 배치 영역에 완전한 무늬 없는 공간은 없다.

장점:

- 새벽과 강이라는 작품의 첫 인상을 가장 즉각적이고 영화적으로 전달한다.
- 우측 빛과 수면 반사가 시선을 끌어 정적인 타이틀에 생명력을 준다.
- 전경·중경·후경이 뚜렷해 향후 미세한 패럴랙스 레이어로 분리하기 좋다.

단점과 UI 영향:

- 우측 새벽빛과 반사가 강해 중앙 정렬 로고·메뉴보다 먼저 시선을 빼앗을 수 있다.
- 중앙 메뉴 영역의 수면 물결과 반사 명도가 위치별로 달라 텍스트 대비가 흔들린다.
- 1280×720 축소 시 좌우 식생의 가는 선과 수면의 고주파 세부가 뭉개지거나 메뉴 주변 노이즈가 될 수 있다.
- 밝은 금색 계열 로고를 사용할 경우 우측 광원과 색·명도가 겹칠 수 있으므로 중앙 스크림이 필수다.

적합한 사용:

- 강한 첫인상과 영화적 분위기가 우선일 때 적합하다.
- 사용할 경우 우측 광원 강도를 낮추고, 중앙 728×486 영역의 수면 세부와 명도 편차를 줄인 UI용 후처리 버전이 필요하다.

### 3.2 키비주얼 02

관찰:

- 좌측의 절제된 새벽빛에서 중앙 강과 우측 협곡으로 이어지는 S자 흐름이 있다.
- 전경은 우측 하단에 집중되고, 중앙 하단 수면은 01보다 넓고 차분하다.
- 배는 좌측 하단에 있어 중앙 UI와 직접 충돌하지 않는다.
- 전체 명도와 채도가 낮고 안개 낀 산맥이 넓게 펼쳐져 있다.

장점:

- 중앙 수면의 세부 밀도가 낮아 메뉴와 도움말 배치에 유리하다.
- 빛이 화면 좌측에 머물러 중앙 로고와 메뉴를 덜 방해한다.
- 강의 굽이가 원경으로 이어져 `떠남과 여정`이라는 전체 서사를 은유한다.
- 중앙 스크림을 얹었을 때 배경의 주요 피사체를 크게 가리지 않는다.

단점과 UI 영향:

- 중앙 상단의 회색 구름과 산맥이 중간 명도라 밝은 로고만 올리면 대비가 충분하지 않을 수 있다.
- 01보다 드라마와 고유한 초점이 약해, 정식 로고와 모션이 없으면 일반적인 산수 풍경으로 보일 위험이 있다.
- 어두운 우측 하단 전경과 중앙 수면의 경계 위에 메뉴가 걸릴 경우 버튼별 배경 명도가 달라질 수 있다.

적합한 사용:

- 타이틀 UI의 안정적 가독성과 긴 체류 시간을 우선할 때 더 적합하다.
- 중앙 스크림과 아주 약한 수면·안개 모션을 더하고, 좌측 새벽빛과 강의 진행 방향을 유지하는 것을 권장한다.

### 3.3 비교 결론

| 항목 | 01 | 02 |
|---|---|---|
| 서사적 첫인상 | 강함: 배·강·극적인 새벽 | 중간: 여정과 공간감 중심 |
| 중앙 UI 가독성 | 보정 필요가 큼 | 상대적으로 유리 |
| 중앙 세부 밀도 | 물결·반사 세부가 많음 | 수면이 넓고 차분함 |
| 시선 경쟁 | 우측 광원이 강하게 경쟁 | 광원이 좌측에 머물러 덜 경쟁 |
| 1280×720 축소 | 가는 전경·물결이 뭉칠 위험 | 형태가 비교적 안정적 |
| 추천 용도 | 오프닝·홍보 키아트 후보 | 타이틀 실사용 기반 후보 |

## 4. 로고 러프 시트 검토

세 가지 러프는 손으로 새긴 듯한 먹·목각 질감과 서로 다른 굵기를 탐색한다. 중간 안은 작은 크기에서도 덩어리가 가장 강하지만 획 내부가 막히고, 위·아래 안은 상대적으로 가볍지만 생성형 질감과 불규칙한 외곽이 남는다.

### 최종 사용 차단 조건

**현재 로고 시트는 정식 타이틀, Steam 캡슐, 홍보물, 게임 내 이미지에 사용할 수 없다(P0).** 생성형 이미지의 한글 획과 자소 구조가 정자 기준으로 안정적으로 보존되지 않았고, 일부 획의 접합·받침·내부 공간이 글자 오류 또는 오자로 인식될 가능성이 있다. 화면에 의도한 문자열이 비슷하게 보이는지만으로 승인하지 않는다.

정식 사용 전 필수 조건:

1. 정확한 Unicode 문자열 `주몽신화전기`를 승인된 한글 글꼴로 먼저 조판한다.
2. 조판을 바탕으로 디자이너가 벡터에서 획을 수작업 정리한다. 생성 이미지의 글자 형태를 그대로 트레이싱하지 않는다.
3. 국문 교정자가 100%, 50%, 25% 크기에서 여섯 글자를 각각 판독하고 서면 승인한다.
4. `주 / 몽 / 신 / 화 / 전 / 기`의 초성·중성·종성, 내부 공간, 획 접합을 글자별 체크한다.
5. 밝은 배경·어두운 배경용 단색 벡터와 1280×720 실제 크기 래스터를 별도 검증한다.

질감 방향은 참고할 수 있으나 글자 원본으로는 사용할 수 없다.

## 5. 위험 분류

### P0 — 최종 사용 차단

- 로고 러프 시트의 한글 획·자소 오류 가능성. 정확한 문자열 조판, 벡터 재작성, 국문 교정 전까지 모든 최종 사용을 차단한다.

### P1 — 높은 가독성·구현 위험

- 두 키비주얼 모두 중앙에 로고·메뉴를 실제 합성한 1280×720 증거가 없다. 명도 대비는 추정만 가능하다.
- 01은 우측 광원과 수면 반사가 UI보다 강하고 중앙 세부 밀도가 높다. 스크림과 국소 톤 보정 없이 사용하면 메뉴 가독성을 해칠 수 있다.
- 02도 중앙 상단이 중간 명도라 밝은 로고 대비가 자동 보장되지 않는다.
- 비활성 `설정`의 사유 안내와 메뉴 전체 상태 시트가 없다.
- 16:9 외 크롭·레이아웃 정책이 시안에 반영되지 않았다.

### P2 — 품질·확장 위험

- 와이어프레임 좌우 안전영역 48px은 5% 기준보다 좁다.
- 버전 표기 13px은 축소·DPI 환경에서 가독성이 낮을 수 있다.
- 01의 전경 가는 식생과 물결, 02의 안개 낀 원경은 1280×720 축소와 압축에서 뭉개질 수 있다.
- 키비주얼 레이어 분리 여부가 확인되지 않아 모션 적용 시 품질과 성능 위험이 있다.
- 포커스의 적동색 마커는 비색상 외곽선과 병행되어 방향은 좋지만, 실제 배경 위 3:1 인접 대비 측정이 필요하다.

## 6. 최종 추천

정식 타이틀의 기반은 **키비주얼 02**를 추천한다. 중앙 UI 영역이 상대적으로 차분하고 광원이 중앙에서 벗어나 있어 장시간 메뉴 가독성에 유리하다. 키비주얼 01은 더 강한 영화적 장면이므로 오프닝 전환 컷, 홍보용 키아트 또는 타이틀 진입 직전 풀스크린 이미지 후보로 보존할 가치가 있다.

추천 조합:

- 배경: 키비주얼 02를 기반으로 중앙 로컬 스크림 적용
- 구도: 현재 와이어프레임의 중앙 정렬 구조 유지
- 안전영역: 핵심 텍스트 좌우 최소 64px, 상하 최소 40px
- 포커스: 밝은 외곽선 + 좌측 형태 마커 + 명도 변화 유지
- 로고: 현재 러프의 질감만 참고하고 정확한 한글 벡터로 전면 재제작

이 추천은 합성·실행 화면을 확인하기 전의 `조건부 추천`이며 최종 승인이 아니다.

## 7. 다음 검증 항목

1. 키비주얼 01·02 각각에 와이어프레임 UI를 실제 합성한 1280×720 비교본을 만든다.
2. 중앙 스크림 유무와 불투명도 2~3단계를 비교한다.
3. 제목, 부제, 활성 메뉴, 비활성 메뉴, 도움말의 최소 대비를 가장 밝고 어두운 프레임에서 측정한다.
4. 포커스 기본·호버·누름·비활성·전환 잠금 상태를 색각 이상 시뮬레이션과 흑백 화면에서 확인한다.
5. 1920×1200(16:10), 2560×1080 및 3440×1440(울트라와이드) 크롭을 확인한다.
6. 창 모드 960×540과 DPI 125%에서 글자와 하단 정보가 잘리지 않는지 확인한다.
7. 정확한 한글 벡터 로고를 100%, 50%, 25% 크기로 국문 교정한다.
8. Godot 적용 후 키보드·마우스·게임패드 포커스와 새 게임→오프닝 전환을 화면·로그로 재검증한다.

- 키비주얼: 02를 타이틀 기반으로 조건부 추천, 실제 UI 합성·대비 측정 미확인.
