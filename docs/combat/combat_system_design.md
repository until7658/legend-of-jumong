# 주몽신화전기 전투 시스템 기획

## 1. 설계 목표

- 4인 파티의 역할과 인물 성격이 전투 행동으로 드러나는 턴제 전투를 만든다.
- 단순 화력 경쟁보다 예약 행동, 공간 소유, 조건 대응, 관계 협동을 중심으로 한다.
- 한 전투는 일반전 3~6분, 정예전 6~10분, 보스전 10~15분을 목표로 한다.
- 이 문서의 수치는 프로토타입용 초기값이며 실제 플레이 테스트 뒤 조정한다.

## 2. 전투 편성 및 전장

- 전투 참가자는 아군 최대 4명, 적 권장 1~6명이다.
- 아군은 `전열` 2칸과 `후열` 2칸을 사용한다. 적도 전열과 후열을 가진다.
- 전열은 근접 공격의 기본 대상이며 후열은 근접 피해 20% 감소를 받는다.
- 후열 앞의 전열이 비면 해당 후열은 `노출`되어 피해 감소를 잃는다.
- 위치 이동은 행동 하나를 소비하지만 일부 스킬은 공격과 이동을 동시에 수행한다.

## 3. 턴 흐름과 상태

전투 상태는 다음 순서를 명시적으로 따른다.

`BATTLE_START → ROUND_START → ORDER_PREVIEW → COMMAND_SELECT → ACTION_RESOLVE → ROUND_END → VICTORY/DEFEAT`

1. `ROUND_START`: 지속 효과, 재사용 대기시간, 라운드 시작 패시브를 처리한다.
2. `ORDER_PREVIEW`: 민첩과 행동 우선도로 이번 라운드의 예상 순서를 공개한다.
3. `COMMAND_SELECT`: 살아 있는 아군 전원의 행동과 대상을 결정한다.
4. `ACTION_RESOLVE`: 확정 순서대로 행동한다. 전투 불능, 차단, 대상 소멸을 즉시 반영한다.
5. `ROUND_END`: 지속 피해, 지속시간 감소, 승패를 판정한다.

행동 순서 점수는 `민첩 + 행동 우선도 + 0~5 난수`로 계산한다. 같은 점수면 아군, 민첩, 편성 순서 순으로 처리한다. 난수 폭을 작게 유지해 순서 예고의 신뢰성을 보장한다.

## 4. 기본 능력치와 공식

| 능력치 | 기능 |
|---|---|
| 생명 | 0이 되면 전투 불능 |
| 힘 | 근접·투척 물리 피해 |
| 기교 | 활·정밀 공격과 상태 적중 |
| 방어 | 물리 피해 경감 |
| 의지 | 상태 저항과 회복 효율 |
| 민첩 | 행동 순서와 회피 |

- 물리 피해: `max(1, 위력 × 공격 능력치 / (방어 + 50))`
- 회복량: `위력 × (100 + 의지) / 100`
- 상태 적중률: `clamp(기본 확률 + 시전자 기교 - 대상 의지, 10, 95)%`
- 치명타 기본 확률은 5%, 치명타 피해는 150%다.
- 최종 피해에는 후열, 방어, 표식, 난이도 등 배율을 곱한다.

## 5. 전투 자원

### 기력

- 캐릭터별 최대 5점이다.
- 전투 시작 시 2점, 기본 공격 시 1점, 방어 시 1점을 얻는다.
- 일반 전술기는 1~2점, 궁극기는 4~5점을 소비한다.
- 기력은 전투가 끝나면 초기화한다.

### 고유 자원

고유 자원은 조작 부담을 줄이기 위해 캐릭터마다 하나만 사용한다.

- 주몽 `호흡`: 사선이 조건대로 발동하거나 발사를 미루면 획득, 최대 3.
- 오이 `충격`: 공격 방향을 비틀거나 피해를 받아 저장, 최대 3.
- 마리 `기세`: 길을 만들거나 같은 길이 반복 이용되면 획득, 최대 3.
- 협보 `근거`: 서로 다른 기록 조건을 적중시키면 획득, 최대 3.
- 소서노 `판돈`: 거점에 행동·기력·피해가 투입되면 획득, 최대 5.
- 파티 `호흡`: 관계 행동으로 서로의 핵심 규칙을 연결하면 획득, 최대 5.

## 6. 핵심 전술 규칙

### 예약과 개입

- 사선, 기록, 처방은 미래 행동에 조건을 거는 `예약`이다.
- 조건이 성립하면 현재 행동 도중 `개입` 상태로 들어가 예약 효과를 해결한 뒤 원래 행동을 계속한다.
- 개입 순서는 생존 보호 → 대상 변경 → 이동 변경 → 공격 → 사후 효과 순이다.
- 한 원본 행동에는 개인 개입 1회와 협동 개입 1회까지만 허용한다.

### 길과 거점

- 길은 두 칸의 이동 관계를 바꾸고, 거점은 한 칸에 투입된 비용을 기록한다.
- 이동을 바꾸는 효과가 충돌하면 가장 최근에 공개된 효과를 우선하되 UI에서 결과를 사전 표시한다.
- 거점의 판돈은 소서노가 결산하거나 거점이 파괴될 때만 소비한다.

### 표식

- 표식 대상은 받는 정밀·투척 피해가 15% 증가한다.
- 하나의 대상에는 표식 하나만 유지되며 기본 지속시간은 2라운드다.

### 엄호

- 지정 아군이 공격받을 때 엄호자가 반응한다.
- 원거리 엄호는 공격자에게 차단 사격, 방패 엄호는 피해 대신 받기로 작동한다.
- 캐릭터마다 라운드당 기본 1회만 발동한다.

### 붕괴

- 강제 이동, 방패 충격, 특정 상태 누적으로 적의 대형을 무너뜨린다.
- 붕괴된 대상은 1라운드 동안 방어가 20% 감소하고 반격할 수 없다.
- 보스는 완전 붕괴 대신 `균형 손상`을 누적하며 3회 누적 시 붕괴한다.

### 연계

- 먼저 적용된 상태와 어울리는 후속 스킬은 추가 효과를 낸다.
- 대표 연계: `노출 → 정밀 사격`, `젖음 → 둔화`, `연막 → 기습`, `붕괴 → 강타`.
- 연계 보너스는 한 행동당 한 번만 적용해 폭발적인 중첩을 막는다.

### 관계 협동

- 관계는 낯섦, 신뢰, 유대, 맹약의 네 단계이며 서사 사건으로 성장한다.
- 협동 반응기·연계기·맹약기는 파티 호흡을 각각 2·3·5 소비한다.
- 협동기는 두 캐릭터의 핵심 규칙이 실제로 결합될 때만 발동할 수 있다.
- 협동 행동은 행동 순서표에 두 인물의 초상과 발동 조건을 함께 예고한다.

## 7. 상태이상 원칙

- 약화 상태는 같은 종류가 중첩되지 않고 지속시간만 긴 값으로 갱신한다.
- 행동 불능 상태는 일반 적 1라운드, 정예·보스에게는 저항 또는 대체 효과를 적용한다.
- 화면에는 상태 아이콘, 남은 라운드, 수치 변화를 함께 표시한다.
- 상태 목록의 기준 데이터는 `data/combat/status_effects.json`을 사용한다.

## 8. 행동과 승패

- 기본 행동: 공격, 스킬, 방어, 이동, 도구, 도주.
- 방어는 다음 자기 차례까지 받는 피해를 35% 줄인다.
- 도구는 파티 공용 재고를 소비하며 캐릭터 스킬을 대체하지 않는다.
- 모든 적을 전투 불능으로 만들면 승리, 전투 필수 파티원이 모두 전투 불능이면 패배다.
- 서사 전투는 생존, 탈출, 특정 대상 보호 등 별도 승리 조건을 허용한다.

## 9. 적 설계 기준

- 일반병: 한 가지 역할과 한 가지 대응 단서.
- 정예병: 두 역할의 조합과 하나의 연계 차단 수단.
- 보스: 2~3단계 행동 변화, 예고된 강공격, 균형 손상 규칙.
- 적의 강한 행동은 순서표, 자세, 대사 또는 바닥 표시로 사전 예고한다.

## 10. 대표 조합 검토

- 추격병 다수: 오이가 전열을 고정하고 마리가 후열을 노출시킨 뒤 주몽이 정밀 사격한다.
- 매복전: 협보의 연막으로 첫 피해를 줄이고 마리가 위치를 바꾸며 주몽이 차단 사격한다.
- 지휘관과 호위병: 오이가 호위 대형을 붕괴시키고 협보가 지휘관을 분석한 뒤 집중 공격한다.

## 11. UI 요구사항

- 항상 행동 순서, 전열·후열, 기력, 고유 자원, 파티 호흡, 상태 지속시간을 표시한다.
- 사선, 공격 방향, 길, 기록, 거점은 색상만이 아니라 서로 다른 선과 문양으로 구분한다.
- 스킬 선택 시 비용, 예상 피해·회복, 연계 결과, 대상 변화가 미리 보여야 한다.
- 전투 로직은 UI와 직접 결합하지 않고 상태 변경과 행동 요청을 시그널로 전달한다.
